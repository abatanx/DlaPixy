# Palette and Color

## Current Palette Features
- Color helpers:
  - `rgbaToHsva`
  - `hsvaToRgba`
- Initial palette is the 216 web-safe colors.
- Palette entries are normalized as `{ id, color, caption, locked }`.
- Caption max length is `100` (`PALETTE_CAPTION_MAX_LENGTH` in `shared/palette.ts`).
- Existing swatches open the color modal on double-click.
- Palette modal edits `#RRGGBB` + separate `AA` input plus RGBA / HSV controls.
- Palette modal preview shows original vs editing color with `Delta HSV`.
- Selected drawing color and palette entries can carry alpha as `#RRGGBBAA`.
- Legacy `#RRGGBB` values are normalized on load.

## Usage Overlay and Removal
- While `Cmd/Ctrl` is held, each swatch shows summarized usage count.
- The selected swatch exposes a jump action to the first matching canvas pixel (`for y` then `for x`) and sets a `1x1` selection there.
- Removing a used swatch requires confirmation and clears matching pixels to transparent before removal.
- Editing an existing palette color also replaces matching canvas pixels in one undoable action.
- Apply is disabled if an edited palette color would duplicate another palette color.
- The final `+` palette cell opens the same modal in create mode for a new unique palette color.

## Merge and Ordering
- `Cmd/Ctrl + click` toggles multi-selection for merge.
- If a normal swatch is already selected, the first `Cmd/Ctrl + click` seeds merge with both swatches.
- Merge UI appears above the workspace canvas column without pushing the swatch grid down.
- The destination swatch is explicitly chosen and marked with a compact `و®‹` badge.
- Merge can also bulk-delete selected swatches using the same clear-if-used behavior.

## Stable Swatch Identity History (`#56`)
- `PaletteEntry.id` is the durable swatch identity.
- IDs are UUID-format strings generated by `crypto.randomUUID()`.
- Sidecar schema was bumped to `2` for palette entry IDs.
- Identity-sensitive UI flows now track swatches by `id`:
  - hover target
  - reference lines
  - merge selection
  - drag-and-drop target
- Color-semantic flows still resolve by `color`:
  - usage counts
  - pixel replacement
  - delete / merge execution
